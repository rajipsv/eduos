Workflow & High-Level Design

EduOS

Tutoring operations platform — not an LMS. Multi-tenant MVP architecture, roles, nine layers, and core workflows.

Product: EduOS (tutor-hub) · Version: local-first demo · 2026

1. What EduOS Is

EduOS is the operating system for tuition centers. It handles everything around teaching:

  • Acquiring and enrolling students (CRM)
  • Running batches, schedules, attendance, and tests
  • Messaging parents and staff
  • Tracking student/tutor success signals
  • Business intelligence and decisions
  • Connecting external tools (Zoom, WhatsApp, calendars)

Teaching itself happens on the tutor's own platforms (Zoom, Google Meet, Khan Academy, etc.). EduOS coordinates logistics and visibility.

2. Design Principles

PrincipleMeaning
Ops-firstCRM, schedule, attendance, comms — not course hosting
Multi-tenantEach center's data isolated by centerId
Role-based portalsSame app, different nav and permissions per role
Decisions, not dashboardsBI suggests what to do next
Local MVPData in browser localStorage; run via start.bat

Deferred: Payments / billing (Layer 10 removed from roadmap).

5. Core Workflows

5.1 Lead → Student (Layer 1)

Website inquiry→ CRM lead→ Pipeline→ Demo→ Counseling→ Batch→ Student

Pipeline stages: Lead → Inquiry → Demo → Counseling → Admission → Payment → Batch allocation → Student

  • Public Website form writes to CRM
  • Tabs: Pipeline, All leads, Follow-ups
  • Log activities, schedule demos, move stages, enroll to batch
  • WhatsApp follow-up from lead detail

5.2 Daily academy operations (Layer 2)

Setup once

Add teachers → Create batches → Topics + days/times → Auto-generate schedule

Each week

Schedule + links → Mark attendance → Notify parents → Record tests → Notify parents

  • One topic = one session; auto-generate Meet/Zoom/Teams links
  • Mark done on completed classes
  • Attendance: present / absent / late → optional parent WhatsApp

6. End-to-End Journey (New Center)

  1. Platform owner registers center
  2. Center admin configures Settings; public site + CRM live
  3. Create batches, generate schedule; link teacher accounts
  4. Leads convert to students
  5. Teachers mark attendance, assign homework
  6. Automations notify parents; parent portal active
  7. Owner uses BI decisions and reports

9. Permission Summary

  • Platform owner — all centers + support view
  • Center admin — all layers in center
  • Teacher — scoped batches/students + homework tools
  • Student — schedule + homework submit
  • Parent — view + message + mark homework submitted

10. Out of Scope (MVP)

  • Payment collection and invoicing
  • Hosting course videos or worksheet libraries
  • Production backend / real multi-user sync
  • Native mobile apps
  • Full API enforcement (demo UI only)
